Closing the gap — three layers

1. Use O_NOFOLLOW on the open
const fd = await fsP.open(dest,
  fs.constants.O_WRONLY |
  fs.constants.O_CREAT  |
  fs.constants.O_TRUNC  |
  fs.constants.O_NOFOLLOW,
  mode);
await fsP.write(fd, x.data);
await fsP.close(fd);
2. Reject duplicate paths up front
const seen = new Set();
for (const f of files) {
  const norm = path.posix.normalize(f.path);
  if (seen.has(norm)) {
    throw new Error('duplicate entry: ' + norm);
  }
  seen.add(norm);
}
3. Serialize entry extraction
// Replace Promise.all(files.map(...))
for (const x of files) {
  await extractOne(x, output, opts);
}

Why all three Each one alone is sufficient for the duplicate-name case. Together they remove the entire class of bug:

(1) prevents any open from following an unexpected symlink — the strongest, smallest patch.

(2) prevents the malicious archive from even being processed.

(3) collapses the inter-chain race into a sequential check-then-use; safer foundation for future changes.
