Where fs calls actually run

JS main thread fsP.readlink(dest) → submit work item, return Promise pending
↓
libuv thread pool 4 worker threads (default UV_THREADPOOL_SIZE=4) executing real syscalls
↓
completion signal worker writes to a pipe / eventfd — main thread wakes up
↓
poll phase callback runs as macrotask — resolves the Promise — drains the .then() chain

Promise chains parallelise across the pool, but each chain's own steps are still serialized: hop N+1 cannot start until hop N's callback runs.
