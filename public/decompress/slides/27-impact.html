Impact — in Node, AFW is RCE

Punchline You don't need ~/.ssh/authorized_keys. You don't need a cron file. You don't need a writable node_modules/.bin. Every running Node process turns arbitrary file write into arbitrary code execution — even on a read-only filesystem.

1. libuv pipe fd anonymous pipe for signal / async wakeups, visible at /proc/<pid>/fd/
2. craft struct uv__signal_msg_t with a fake uv_signal_s handle and chosen signum
3. libuv reads next event-loop iteration dispatches “your signal”
4. signal_cb fires function pointer in the fake handle → ROP gadget → RCE

Why it works in Node Official builds ship as non-PIE — the Node segment is loaded at a fixed, known address, so ROP gadgets and the address of uv_signal_s-shaped data are predictable to the attacker.
