The bug, in one line

vulnerable — input INTO the template source
@app.route('/hello')
def hello():
    name = request.args.get('name', '')
    template = f"<h1>Hi {name}</h1>"
    return render_template_string(template)

# /hello?name={{7*7}}
# → renders <h1>Hi 49</h1>
safe — input STAYS in the context
@app.route('/hello')
def hello():
    name = request.args.get('name', '')
    return render_template_string(
        "<h1>Hi {{ name }}</h1>",
        name=name,
    )

# /hello?name={{7*7}}
# → renders <h1>Hi {{7*7}}</h1>

The rule If user input is concatenated into the template source, you have SSTI. If it's bound as a context variable, you don't.

Spotting it Look for f"...", "..." + name, format(), %s, or any string assembly that ends up at render_template_string, Twig::render, FreeMarker.process, etc.

Variant Sometimes the template is loaded from a DB / CMS where users edit it — same bug, different facade.
