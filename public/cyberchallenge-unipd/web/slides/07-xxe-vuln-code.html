The vulnerable code — three flavors

PHP — libxml < 2.9
$xml = file_get_contents('php://input');
$doc = simplexml_load_string($xml);
echo $doc->name;

// Old libxml: external entities ON by default
// libxml_disable_entity_loader(false) re-enables it
Java — DocumentBuilder default
DocumentBuilderFactory f =
    DocumentBuilderFactory.newInstance();
DocumentBuilder db = f.newDocumentBuilder();
Document d = db.parse(req.getInputStream());

// Pre-JAXP-1.5 / older containers: XXE ON
Python — lxml with resolve_entities=True
from lxml import etree

parser = etree.XMLParser(
    resolve_entities=True,    # explicit opt-in
    no_network=False,         # allows http://
)
doc = etree.parse(req.body, parser)

Pattern Every language ships an XML parser. Every one of them has had unsafe defaults at some point. Newer libs default-deny, older deployments don't.

In CTFs When the challenge accepts Content-Type: text/xml or application/xml, try the entity payload first — before anything else.
