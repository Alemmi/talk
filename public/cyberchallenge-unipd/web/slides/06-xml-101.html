XML 101 — the feature that bites

internal entity — harmless
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY greet "ciao">
]>
<msg>&greet;</msg>

<!-- parses to: <msg>ciao</msg> -->
external entity — the parser fetches
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<msg>&xxe;</msg>

<!-- parses to: <msg>root:x:0:0:...</msg> -->
  • ▸
    DTD — Document Type Definition, declares what's allowed in the document. Can be inline (<!DOCTYPE foo [...]>) or external
  • ▸
    Entity — named macro: &greet; expands to its definition
  • ▸
    SYSTEM identifier — tells the parser to resolve the entity by fetching a URI: file://, http://, ftp://, sometimes jar:, netdoc:, php://
  • ▸
    This is a feature. The bug is feeding it untrusted XML
