Closing the gap — disable, don't filter

Java — harden every factory
DocumentBuilderFactory f =
    DocumentBuilderFactory.newInstance();
f.setFeature(
  "http://apache.org/xml/features/" +
  "disallow-doctype-decl", true);
f.setFeature(XMLConstants
  .FEATURE_SECURE_PROCESSING, true);
f.setXIncludeAware(false);
f.setExpandEntityReferences(false);
Python — defusedxml or strict parser
from defusedxml import ElementTree as ET
doc = ET.fromstring(req.body)

# or with lxml:
parser = etree.XMLParser(
    resolve_entities=False,
    no_network=True)
PHP — reject DOCTYPE
// Modern libxml: external entities OFF by default.
// Make it explicit anyway:
$prev = libxml_use_internal_errors(true);
$doc = new DOMDocument();
$doc->loadXML($xml,
    LIBXML_NONET | LIBXML_DTDLOAD ^ LIBXML_DTDLOAD);
// Or simply reject any input containing <!DOCTYPE

Layer 1 Disable DTDs and external entities at the parser. Don't try to filter strings — entity syntax has too many tricks (UTF-7, parameter entities, error-based exfil).

Layer 2 Prefer JSON. If you must accept XML, accept it from one place and harden one parser.

Layer 3 Egress filtering. The parser pod doesn't need outbound HTTP.
