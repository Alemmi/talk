Template engines 101

a Jinja2 template — intended use
<h1>Hi {{ user.name }}</h1>
<p>You have {{ user.posts|length }} posts.</p>
{% for p in user.posts %}
  <li><a href="/p/{{ p.id }}">{{ p.title }}</a></li>
{% endfor %}
render call — the safe way
render_template_string(
    template_source,    # constant or from a file
    user=current_user,  # data goes in the context
)
  • ▸
    The contract: you pass a template + a context; you get back a string
  • ▸
    The engine resolves attribute access — user.posts|length walks objects, calls methods, runs filters
  • ▸
    Many engines run arbitrary code by design — Jinja {% if %}, ERB <% ... %>, JSP scriptlets
  • ▸
    The line between “data access” and “arbitrary code” is the engine's sandbox — and most don't sandbox by default
  • ▸
    The big ones: Jinja2, Twig, Freemarker, Velocity, Thymeleaf, ERB, Mustache, Smarty, Handlebars, Pug
