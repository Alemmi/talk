Takeaways

  • ▸
    Untrusted input + powerful interpreter = primitive. The interpreter doesn't need a CVE — it's doing exactly what it was designed to do
  • ▸
    XXE: every XML parser is a network and file client. If you can't disable DTDs, treat any XML endpoint like a Trojan horse — and probe for SSRF, not just file read
  • ▸
    SSTI: keep user input in the context, never in the template source. Concatenation is the bug, parameter binding is the fix
  • ▸
    7×7 is your first probe. Then fingerprint the engine, then pick the gadget — same idea, different syntax
  • ▸
    Filtering is a losing game. Disable the dangerous feature at the parser/engine; don't try to outsmart entity syntax or template DSLs with regex
  • ▸
    In CTFs — check every input. URL params, headers, JSON, filenames, error messages, server-generated emails. The flag is wherever the dev forgot the parser was loaded
