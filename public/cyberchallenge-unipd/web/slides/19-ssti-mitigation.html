Closing the gap — bind, don't concatenate

1. User input goes in the context, never in the source
# BAD
template = f"Hello {name}"
render_template_string(template)

# GOOD
render_template_string(
    "Hello {{ name }}", name=name)
2. Sandboxed renderer (best-effort)
from jinja2.sandbox import SandboxedEnvironment
env = SandboxedEnvironment()
env.from_string(tpl).render(**ctx)
# Twig: new \Twig\Sandbox\SecurityPolicy(...)
3. Least-privilege rendering worker
  • No DB credentials in the rendering process
  • No outbound network from the renderer
  • Read-only filesystem; no shell available
  • Short-lived process, killed after each request

Why “all three” Sandboxes have a long history of bypasses (Jinja sandbox CVEs, Twig sandbox CVEs, Velocity escapes). Rule (1) is the only real fix; (2) and (3) buy you time when (1) is hard to retrofit.

Anti-pattern “Block {{ in input.” You'll miss the alternate syntaxes, the URL-encoded variants, the |attr bypass, and the engine that uses ${.
