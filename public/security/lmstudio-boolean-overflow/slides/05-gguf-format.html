GGUF — a bag of key/value metadata

the parts that matter to us
┌─ header ─────────────────────┐
│ magic "GGUF", version, counts │
├─ KV metadata ────────────────┤
│ general.architecture = "..."  │  ◄ picks the code path
│ .block_count    = u32   │  ◄ becomes n_layer
│ .attention.*    = ...   │
│ tokenizer.ggml.model  = "..." │
│ tokenizer.ggml.tokens = [...] │
│ ...arbitrary KV strings...    │
├─ tensor info + tensor data ──┤
└──────────────────────────────┘

Every value above is read verbatim from the file at load time, before any inference runs.

Typed arrays KV values can be typed arrays — UINT8, INT32, strings… We will lean on UINT8 arrays later to smuggle bytes with embedded NULs.

Trust model The loader treats counts and lengths as facts, not as untrusted input. That assumption is the whole bug.
