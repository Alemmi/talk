CWE-787 The loop runs il = 0 … n_layer but the array stops at 512. So n_layer is a length dial: every unit above 512 is one more 32-bit write past the end, all attacker-controlled.

What value we can write Each word is the boolean il % swa_period < (swa_period−1) — only ever a 0 or a 1, never an arbitrary byte. With swa_period = 6 it is the repeating stencil 1 1 1 1 1 0: a 0 on every 6th slot, a 1 elsewhere.

So how long should it be? Exactly as long as it takes to reach the field we want to corrupt — not a digit more. We can't answer that yet: we have to see what lives after the array first. That is the next section — and it is what fixes the concrete n_layer = 3852.
