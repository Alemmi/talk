Takeaways

  • ▸
    Model files are untrusted input. A .gguf is data parsed by a C++ engine — treat it like a malicious image or PDF, not like a config you trust
  • ▸
    Every count from the file needs a bound. One missing GGML_ASSERT on n_layer — while its sibling n_expert had one — was the entire root cause
  • ▸
    Adjacency turns OOB into pointer control. A neighbouring unique_ptr at a constant offset means no info-leak is needed — just two 1s in the right place
  • ▸
    Destructors are reachable code. Forcing an exception after corruption turns a quiet OOB write into an immediate, exception-driven hijack
  • ▸
    Non-PIE removes the hard part. Fixed gadget addresses + a saturating heap spray make the exploit deterministic enough to brute-force for free
