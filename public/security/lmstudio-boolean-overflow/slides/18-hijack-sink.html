The hijack — where free_buffer comes from

ggml/src/ggml-backend.cpp
void ggml_backend_buffer_free(
        ggml_backend_buffer_t buffer) {
  if (buffer == NULL) return;          // ◄ 1st: NULL → no-op
  if (buffer->iface.free_buffer != NULL)
    buffer->iface.free_buffer(buffer); // ◄ 2nd: read field, call it
  delete buffer;
}

how we control free_buffer — without ever overwriting it
  • the bufs vector makes ggml_backend_buffer_free run twice; both buffer pointers come from the corrupted bufs, so they point into memory we filled.
  • buffer #1 = NULL → the check returns. It must be NULL, else it derefs garbage and crashes before the pivot.
  • buffer #2 points at our bytes, so buffer->iface.free_buffer is read from them (offset +8) = the gadget 0x236f81c. We never write the field — we own the memory it's read from.
  • the indirect call buffer->iface.free_buffer(buffer) then jumps there → we own RIP. No vtable corruption needed.
