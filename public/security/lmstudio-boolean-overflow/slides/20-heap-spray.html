the only address we must predict
  • Engine is non-PIE → every gadget address is fixed
  • The one unknown: where vocab.pimpl = 0x100000001 actually points
  • Don't predict it — make it true by covering the range
spray parameters
  • Target address — 0x100000001
  • ~4 GB sprayed — ~64,000 GGUF string KVs of 65,504 bytes
  • Periodic 14-slot pattern (14 × 8 bytes), rotated per string
  • Single-shot success 1/14 ≈ 7% — reload is free, fully repeatable

The 14 slots encode, at fixed offsets from 0x100000001:

  • ▸
    the two pointers the destructor consumes — NULL, then a stack-pivot address
  • ▸
    the four-entry argv array for execvp
  • ▸
    the literal /bin/sh\0
  • ▸
    filler cells so any std::string-shaped reads pass minimal sanity

Why UINT8 arrays Gadget addresses contain NUL bytes; a string KV would truncate. UINT8 arrays preserve every byte.
