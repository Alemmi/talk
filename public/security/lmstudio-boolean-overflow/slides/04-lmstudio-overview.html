LM Studio vs llama.cpp — the architecture

LM Studio desktop app + headless / API server
↓
Node-based runtime loads the llm-engine package
↓
llama.cpp engine C++ inference backend — parses the model
↓
GGUF model file attacker-supplied bytes

LM Studio The product: a desktop app to download and run local LLMs, with a GUI and an OpenAI-compatible server. It is the wrapper users interact with.

llama.cpp The open-source C++ engine that actually loads and runs the model. LM Studio bundles it inside llm-engine. The bug lives here — so it affects far more than LM Studio.

Platform Linux x86_64, the non-PIE engine binary bundled at the time of the entry.
