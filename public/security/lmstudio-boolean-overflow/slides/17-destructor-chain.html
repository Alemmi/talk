The destructor walks attacker pointers

src/llama-model.cpp
llama_model::~llama_model() {
  for (auto * lora : loras) {   // iterates corrupted buckets
    delete lora;                // dispatch on attacker pointer
  }
}
src/llama-adapter.h
struct llama_adapter_lora {
  llama_model * model = nullptr;
  // ...
  std::vector<ggml_backend_buffer_ptr> bufs; // each deleter → ggml_backend_buffer_free
  ~llama_adapter_lora() = default;
};
~llama_model iterates corrupted loras
↓
delete lora lora * is attacker-shaped
↓
~llama_adapter_lora destroys the bufs vector
↓
ggml_backend_buffer_free per unique_ptr deleter

The same OOB loop spilled into loras, so the set yields attacker-shaped lora * — and the chain of destructors marches straight to a free.
